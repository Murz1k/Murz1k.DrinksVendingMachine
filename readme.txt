Используемое ПО/Технологии:
Visual Studio Community 2015
MS SQL Express 2014 (localdb)
ASP.NET MVC 5.2
Entity Framework 6
Ajax
AngularJS 1.5

Выполненные требования:
-Вносить сумму, щелкая на кнопки с номиналом монет (1, 2, 5, 10).
-Если монета заблокирована, подсвечивать соответствующую кнопку и блокировать нажатие.
-Показывать внесенную сумму.
-Выбирать напиток, щелкнув на соответствующую картинку.
-При этом, количество оставшихся напитков должно уменьшиться на единицу, а из внесенной суммы должна
быть вычтена стоимость напитка.
-Не позволять выбирать закончившиеся напитки.
-Не позволять выбирать напитки, стоимость которых превышает внесенную сумму.
-Получать оставшуюся сумму в виде сдачи.
-Администрировать ассортимент напитков: добавление, удаление напитков, изменение их количества и стоимости.
-Доступ в административный интерфейс по секретному ключу, который передаётся как параметр в адресной строке.

Невыполненные требования:
-При возврате сдачи показывать количество и номинал монет.
-Возможность импорта напитков.

Controllers/AdminController.cs - контроллер для страницы администратора. Содержит CRUD методы для монет и напитков.
Controllers/HomeController.cs - контроллер для страницы пользователя. Содержит методы для покупки и показа напитков.
Controllers/AdminController.js - angularJs-контроллер стрницы администратора. Содержит CRUD методы для монет и напитков, обновление данных на странице.
Controllers/MachineController.js - angularJs-контроллер стрницы пользователя.
Models/BeverageModel.cs - содержит классы Coin, Beverage, DrinksVendingContext.
Services/BeverageService.cs - обертка DBContext для напитков.
Services/CoinService.cs - обертка DBContext для монет.
Utilities/IoCProvider.cs - утилита для иницилиазиции и использования классов из одной точки.
Utilities/AdminAccessAttribute.cs = атрибут, позволяющий заходить в кабинет администратора только с помощью секретного ключа : 6396482g35ghlh5gl384

Описание функционала: Пользовательский режим:
Страница разделена на 2 части:
	1) Поле выбора напитка. Здесь отображены все выставленные напитки. При наведении на любой из них можно услышать щелчок. Если напитка нет в наличии,
или клиенту не хватает денег, иконка напитка будет неактивна и не кликабельна.
	2) Панель управления своими деньгами. Здесь можно имитировать вроброс монеток, увидеть сколько денег клиент уже запустил в автомат. Имеется кнопка "Сдача",
нажав на которую всплывет окно с отображением сдачи клиента, после чего деньги на счету обнулятся.

Описание функционала: Административный режим:
При первом заходе по этой ссылке: http://localhost:55611/Admin/Index/?secretKey=6396482g35ghlh5gl384 в статическую переменную заносится время входа в панель администратора,
через 5 минут доступ к функциям администратора закроется, поэтому придется входить под ключем еще раз. 
Страница разделена на 3 части: 
	1) Слева панель управления монетами. Для изменения или удаления монеты нужно выбрать одну из существующих, далее нажать на удалить/изменить
и подтвердить в всплывающем окне операцию. Для добавления новой монеты необходимо нажать кнопку "Очистить", далее ввести данные о новой монете в поля и нажать "Добавить".
	2) По середине набор добавленных напитков. Здесь отображены все добавленные напитки. Для осуществления операций над ними необходимо выбрать один напиток и
продолжить действия в правой панели.
	3) Справа панель управления напитками. Панель напитков аналогична панели монет.
P.S. Данные обновляются сразу, но сохраняются в базе при обновлении/закрытии страниц Администратора и Пользователя. Было решено сделать таким образом, так как при частых попытках вносить
изменения в базу возникали ошибки.
